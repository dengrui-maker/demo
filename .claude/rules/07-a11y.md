# 无障碍（Accessibility）规范

## 标准要求

遵循 **WCAG 2.2 AA** 级别标准。

## 语义化 HTML

优先使用语义化元素，而非 `div`/`span` 加事件：

```tsx
// ✅ 正确
<button onClick={handleDelete}>删除</button>
<nav aria-label="主导航">...</nav>
<main>...</main>
<article>...</article>

// ❌ 避免
<div onClick={handleDelete} role="button">删除</div>
```

## 图片与媒体

- 有意义的图片必须有 `alt` 文本
- 装饰性图片使用空 `alt=""`：

```tsx
<img src="/logo.png" alt="公司 Logo" />        // ✅ 有意义
<img src="/divider.png" alt="" role="presentation" />  // ✅ 装饰性
```

## 表单无障碍

每个表单控件必须关联标签：

```tsx
// ✅ 使用 htmlFor 关联
<label htmlFor="email">邮箱地址</label>
<input id="email" type="email" aria-describedby="email-hint" />
<span id="email-hint">请输入有效的邮箱格式</span>

// ✅ 错误信息关联
<input aria-invalid={!!error} aria-describedby={error ? 'email-error' : undefined} />
{error && <span id="email-error" role="alert">{error}</span>}
```

## 键盘导航

- 所有交互元素必须可通过 Tab 键访问
- 自定义组件需正确管理焦点（模态框打开/关闭时）
- 不要移除 `:focus-visible` 样式

```css
/* ✅ 保留焦点指示器 */
:focus-visible {
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
}
```

## 色彩对比

- 正文文字对比度 ≥ 4.5:1
- 大号文字（18px+ 或 14px+ 粗体）对比度 ≥ 3:1
- 不要仅用颜色传递信息（添加图标或文字说明）

## ARIA 使用原则

- 优先使用语义化 HTML，ARIA 是最后手段
- 动态内容变更使用 `aria-live`：

```tsx
<div aria-live="polite" aria-atomic="true">
  {statusMessage}
</div>
```

- 禁止滥用 `aria-label` 覆盖可见文本（两者应保持一致）
